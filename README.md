# Настройка системы (sid)

[![Infostart](https://infostart.ru/bitrix/templates/sandbox_empty/assets/tpl/abo/img/logo.svg)](https://infostart.ru/1c/tools/2741448/)

**Публикация на Infostart:** https://infostart.ru/1c/tools/2741448/

Расширение конфигурации 1С:Предприятие для хранения произвольных настроек системы **без изменения основной конфигурации**.

Подходит, когда нужно быстро добавить «ключ — значение» для интеграций, регламентных заданий, обменных сценариев и прочих точек, где править типовую конфигурацию нежелательно или невозможно.

**Имя расширения:** `инт_НастройкиСистемы`  
**Префикс:** `инт_`  
**Поставщик:** sid  
**Версия:** 1.0.0  
**Режим совместимости:** 8.3.22  
**Репозиторий:** [github.com/sekasid/1c-settings-store](https://github.com/sekasid/1c-settings-store)

---

## Возможности

- Хранение настроек в регистре сведений с ключом — элементом справочника «Раздел настроек»
- Типы значений:
  - строка (до 150 символов)
  - длинная строка (неограниченная)
  - число (15.2)
  - булево
  - дата
  - ссылка на **любой** справочник конфигурации
  - хранилище значений (структура / соответствие / массив и другие сериализуемые типы)
- Мастер создания и изменения настроек
- При создании — автоматически создаётся элемент справочника раздела и подставляется в форму
- Конструктор плоских коллекций для хранилища значений
- Просмотр содержимого хранилища из формы списка / записи
- Программный API чтения и записи (в привилегированном режиме)

## Состав расширения

| Объект | Назначение |
|--------|------------|
| Справочник `инт_РазделНастроек` | Программный ключ настройки (наименование уникально) |
| Регистр сведений `инт_НастройкиСистемы` | Хранение значения |
| Обработка `инт_РедакторНастройкиСистемы` | Мастер создания / изменения |
| Подсистема `инт_ПодсистемаРазработчика` | Командный интерфейс |

### Ресурсы регистра

| Ресурс | Когда использовать |
|--------|-------------------|
| `Значение` | Строка, число, булево, дата, ссылка на справочник |
| `ЗначениеСтрокаНеограниченная` | Длинный текст |
| `ЗначениеХранилище` | Структура, соответствие, массив и др. |

Одновременно заполняется один «логический» канал значения (как в мастере редактора).

## Установка

1. Скачайте cfe-файл расширения (с Infostart) **или** загрузите исходники из этого репозитория в конфигуратор как расширение.
2. Подключите расширение к информационной базе.
3. Убедитесь, что режим совместимости расширения не ниже **8.3.22**.
4. После обновления конфигурации БД объекты появятся в подсистеме **«Подсистема разработчика»**.

Расширение рассчитано на управляемое приложение (Такси).

## Работа в интерфейсе

### Создать настройку

1. Откройте обработку **«Редактор настройки системы»**.
2. Выберите **«Создать настройку»**.
3. Укажите **наименование нового раздела** — будет создан элемент справочника `инт_РазделНастроек`.
4. На следующей странице раздел уже подставлен (только просмотр).
5. Выберите тип значения, заполните данные и нажмите **«Сохранить»**.

### Изменить настройку

1. Выберите **«Изменить настройку»**.
2. Укажите существующий раздел настроек.
3. Отредактируйте тип / значение и сохраните.

### Хранилище значений

Два способа ввода:

- **Текст** — результат `ЗначениеВСтрокуВнутр` (не JSON).
- **Конструктор** — плоская Структура / Соответствие / Массив; типы строк: строка, число, булево, дата, справочник, сериализованный текст. Вложенные структуры в конструкторе не создаются.

## Программный интерфейс

Все методы — в модуле менеджера регистра `РегистрыСведений.инт_НастройкиСистемы`.  
Выполняются в **привилегированном режиме** (административный доступ к настройкам из кода без отдельных прав на объекты расширения).

### Прочитать настройку

```bsl
ПараметрыПолучения = Новый Структура;
ПараметрыПолучения.Вставить("РазделНастроек", "МояНастройка"); // имя или ссылка
ПараметрыПолучения.Вставить("ПолучитьЗначение", Истина);

Значение = РегистрыСведений.инт_НастройкиСистемы.ПолучитьНастройкуСистемы(ПараметрыПолучения);

Если Значение = Неопределено Тогда
    // раздел не найден / запись отсутствует / ошибка параметров
КонецЕсли;
```

Можно запросить несколько ресурсов сразу — тогда вернётся структура с ключами запрошенных полей:

```bsl
ПараметрыПолучения = Новый Структура;
ПараметрыПолучения.Вставить("РазделНастроек", РазделСсылка);
ПараметрыПолучения.Вставить("ПолучитьЗначение", Истина);
ПараметрыПолучения.Вставить("ПолучитьЗначениеХранилище", Истина);
ПараметрыПолучения.Вставить("ПолучитьЗначениеСтрокаНеограниченная", Истина);

Данные = РегистрыСведений.инт_НастройкиСистемы.ПолучитьНастройкуСистемы(ПараметрыПолучения);
```

### Записать настройку

```bsl
СтруктураНастройки = Новый Структура;
СтруктураНастройки.Вставить("РазделНастроек", РазделСсылка);
СтруктураНастройки.Вставить("Значение", "текст");
СтруктураНастройки.Вставить("ЗначениеСтрокаНеограниченная", "");
СтруктураНастройки.Вставить("ЗначениеХранилище", Новый ХранилищеЗначения(Неопределено));

Результат = РегистрыСведений.инт_НастройкиСистемы.ЗаписатьНастройкуСистемы(СтруктураНастройки);

Если Не Результат.Успех Тогда
    // Результат.ОписаниеОшибки
КонецЕсли;
```

### Найти / создать раздел по имени

```bsl
// только найти
Раздел = РегистрыСведений.инт_НастройкиСистемы.ПолучитьРазделНастроекПоИмени("МояНастройка");

// найти или создать
Раздел = РегистрыСведений.инт_НастройкиСистемы.ПолучитьРазделНастроекПоИмени("МояНастройка", , Истина);
```

### Прочее

- `ЕстьСодержимоеХранилища(РазделНастроек)` — есть ли непустое значение в хранилище
- `ПолучитьТекстСодержимогоХранилища(РазделНастроек)` — текстовое представление для просмотра

## Важные замечания

- Наименование раздела — **программный ключ**, должно быть уникальным.
- API записи возвращает структуру `{Успех, ОписаниеОшибки}`, а не булево.
- При ошибке чтения / отсутствии записи `ПолучитьНастройкуСистемы` возвращает `Неопределено`.
- Расширение не меняет объекты основной конфигурации — только добавляет свои.
